      #Static Site Search index format Scraps can build a search index using the Fuse JSON schema as shown below. [ { "title": "Search", "url": "http://127.0.0.1:1112/scraps/search.html" }, { "title": "Overview", "url": "http://127.0.0.1:1112/scraps/overview.html" }, ... ] Search libraries Scraps content perform searches with fuse.js using an index. We are considering WASM solutions like tinysearch for future performance improvements in our deployment environment. Configuration If you are not using the search function, please modify your Config.toml as follows.       Setup LSP
      #Integration Currently, the most recommended LSP for editing Scraps is markdown-oxide. markdown-oxide supports the following editing environments: Neovim VSCode Zed Helix To match the current features provided by Scraps, place the following configuration file .moxide.toml under the scraps/ directory and open the scraps/ directory directly for a comfortable editing experience. heading_completions = false title_headings = false tags_in_codeblocks = false references_in_codeblocks = false We are considering a feature to generate the LSP configuration file during the init command.

      MCP Serve
      #CLI ‚ùØ scraps mcp serve This command starts an MCP (Model Context Protocol) server that enables AI assistants like Claude Code to directly interact with your Scraps knowledge base. Examples # Basic MCP server ‚ùØ scraps mcp serve # Serve from specific directory ‚ùØ scraps mcp serve --path /path/to/project The MCP server provides tools for AI assistants to search through your content and list available tags, enabling intelligent assistance with your documentation. For more details, see Integrate with AI Assistants.

      Tag Link
      #Wiki-Links #Markdown If there is no scrap with the specified title, such as #Markdown, it becomes a tag. Tags are displayed on the index page. Each tag links to a page that lists all scraps using that tag.

      #Wiki-Links #Markdown In cases where the same term exists in different contexts and Scrap titles would overlap, you can use the context feature by separating them with folders.
